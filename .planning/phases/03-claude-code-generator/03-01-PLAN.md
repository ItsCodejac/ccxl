<objective>
Create the generator architecture and settings.json + permissions generator that produces fine-grained permission rules matched to the detected project stack.

Purpose: settings.json is the core config file. Permissions must be tuned to the project — a React project needs different bash permissions than a Go project. This plan establishes the generator interface used by all subsequent plans.
Output: Generator interface, settings.json generator with project-aware permissions.
</objective>

**Tech stack:** TypeScript 5.5, zod 3, fs-extra 11
**Established patterns:** Zod schema-first, signal-based registries, barrel exports

**Claude Code settings.json format (from research):**
- permissions.allow/ask/deny arrays with Tool(specifier) syntax: Bash(cmd:*), Read(glob), Edit(glob), WebFetch(domain:x)
- env object for environment variables
- hooks object with event types (PreToolUse, PostToolUse, etc.)
- model, effortLevel, statusLine fields

  <files>src/generators/settings.ts, src/generators/__tests__/settings.test.ts</files>
  <action>
  Create src/generators/settings.ts implementing Generator interface:

  Generates .claude/settings.json with:

  1. Permission rules based on detected stack:
     - Always allow: Read, Glob, Grep, Edit(src/**), WebFetch, WebSearch
     - JavaScript/TypeScript: Bash(npm run *), Bash(npx *), Bash(node *)
     - If yarn detected: Bash(yarn *)
     - If pnpm detected: Bash(pnpm *)
     - If bun detected: Bash(bun *)
     - Python: Bash(python *), Bash(pip *), Bash(pytest *)
     - Go: Bash(go *), Bash(golangci-lint *)
     - Rust: Bash(cargo *)
     - Java: Bash(mvn *), Bash(gradle *)
     - Git always: Bash(git *)
     - Always ask: Bash(git push *), Bash(git reset *)
     - Always deny: Bash(rm -rf *), Read(.env*), Edit(.env*)

  2. Environment variables:
     - NODE_ENV=development (if JS/TS detected)

  3. Model defaults (leave as comment/suggestion, don't override user choice)

  Write tests:
  - JS/TS project → npm/npx/node in allow
  - Python project → python/pip/pytest in allow
  - Go project → go in allow
  - Multi-language → all relevant permissions included
  - .env always denied
  - git push always in ask

  Register generator in pipeline.
  </action>
  <verify>npm run test:run passes, npm run build succeeds</verify>
  <done>settings.json generator produces project-aware permissions, tests verify per-language rules</done>
</task>
