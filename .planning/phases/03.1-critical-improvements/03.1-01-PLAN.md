  Add new hook templates to HOOK_TEMPLATES array:

  1. SessionStart hook — "context-loader.sh":
     - Reads CLAUDE.md and outputs a reminder to follow project instructions
     - Checks for .claude/skills/ and lists available skills
     - Checks for .claude/agents/ and lists available agents
     - Output format: JSON { "hookSpecificOutput": { "message": "..." } }
     - This ensures every new session starts with full context

  2. PreCompact hook — "reread-context.sh":
     - Before context compaction, re-reads critical files
     - Outputs key sections from CLAUDE.md (overview, standards)
     - Lists skills and agents available
     - This prevents context loss during compaction

  Both hooks are UNIVERSAL (no condition — always generated).

  Script logic:
  ```bash
  #!/bin/bash
  MSG=""
  if [ -f "CLAUDE.md" ]; then
    MSG="$MSG\n[CLAUDE.md loaded - follow project instructions]\n"
  fi
  if [ -d ".claude/skills" ]; then
    SKILLS=$(ls .claude/skills/ 2>/dev/null | tr '\n' ', ')
    MSG="$MSG\nAvailable skills: $SKILLS\n"
  fi
  if [ -d ".claude/agents" ]; then
    AGENTS=$(ls .claude/agents/ 2>/dev/null | sed 's/\.md$//' | tr '\n' ', ')
    MSG="$MSG\nAvailable agents: $AGENTS\n"
  fi
  echo "{\"hookSpecificOutput\":{\"message\":\"$MSG\"}}"
  ```

  Add tests:

  - reread-context.sh always generated
  - hooks.json includes SessionStart and PreCompact events
  - Scripts have correct JSON output format
