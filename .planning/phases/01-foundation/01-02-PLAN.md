<task type="auto">
  <name>Task 1: Define CLI command structure with Commander</name>
  <files>src/cli.ts, src/commands/init.ts, src/commands/generate.ts, src/commands/install.ts, src/commands/doctor.ts, src/commands/registry.ts</files>
  <action>
  Build src/cli.ts as the main CLI entry point using Commander:

  Commands (each in src/commands/ as a separate module):
  - `ccxl init` — Interactive project setup (scan + generate). The main command.
    Flags: --dry-run, --yes (skip prompts), --quiet, --verbose, --force (overwrite existing)
  - `ccxl generate` — Generate specific config layers
    Subcommands: settings, skills, hooks, agents, mcp, claude-md, keybindings, all
    Flags: --dry-run, --preview, --target (claude|cursor|copilot|windsurf|all)
  - `ccxl install <package>` — Install from community registry
    Flags: --global, --preview
  - `ccxl doctor` — Run diagnostics and health checks
    Flags: --fix (auto-fix issues), --verbose
  - `ccxl registry` — Browse and manage registry
    Subcommands: search, browse, publish, list
  - `ccxl update` — Check for config drift and suggest updates
    Flags: --apply, --dry-run
  - `ccxl config` — Manage ccxl's own config
    Subcommands: show, set, reset

  Global flags on the program:
  - --version, --help (built-in)
  - --no-color (disable chalk)
  - --debug (verbose logging)

  Each command module exports a function that receives the Commander Command object and attaches its definition. The action handlers should be placeholder stubs that print "Not implemented yet — coming in Phase X" with the relevant phase number.

  Import and register all commands in cli.ts. Use createRequire or read package.json for version.
  </action>
  <verify>npm run build succeeds, node dist/cli.js --help shows all commands, node dist/cli.js init --help shows init flags, node dist/cli.js generate --help shows subcommands</verify>
  <done>All commands registered with help text, all flags defined, stub handlers print phase info, build succeeds</done>
</task>
