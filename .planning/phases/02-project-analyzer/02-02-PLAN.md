<feature>
  <name>Language and Framework Detection</name>
  <files>src/analyzer/detectors/language.ts, src/analyzer/detectors/framework.ts, src/analyzer/detectors/__tests__/language.test.ts, src/analyzer/detectors/__tests__/framework.test.ts</files>
  <behavior>
  Language detector:
  - package.json present → detects "javascript" (or "typescript" if tsconfig.json exists)
  - pyproject.toml or requirements.txt → detects "python"
  - go.mod → detects "go"
  - Cargo.toml → detects "rust"
  - pom.xml or build.gradle → detects "java"
  - Package.swift → detects "swift"
  - Multiple config files → detects all languages
  - No config files → returns empty array

  Framework detector (reads package.json dependencies + other signals):
  - react in deps → { name: "react", category: "frontend" }
  - next in deps → { name: "next", category: "fullstack" }
  - vue in deps → { name: "vue", category: "frontend" }
  - nuxt in deps → { name: "nuxt", category: "fullstack" }
  - svelte in deps → { name: "svelte", category: "frontend" }
  - angular/core in deps → { name: "angular", category: "frontend" }
  - express in deps → { name: "express", category: "backend" }
  - fastify in deps → { name: "fastify", category: "backend" }
  - prisma in deps → { name: "prisma", category: "orm" }
  - drizzle-orm in deps → { name: "drizzle", category: "orm" }
  - Django in pyproject.toml → { name: "django", category: "backend" }
  - Flask in pyproject.toml → { name: "flask", category: "backend" }
  - FastAPI in pyproject.toml → { name: "fastapi", category: "backend" }
  - vite in deps → { name: "vite", category: "build" }
  - jest/vitest/mocha in devDeps → detect as test frameworks

  Package manager detector:
  - package-lock.json → npm
  - yarn.lock → yarn
  - pnpm-lock.yaml → pnpm
  - bun.lockb → bun

  Use mock file systems in tests (create temp dirs with specific files). Do NOT use actual project files.
  </behavior>
  <implementation>
  Each detector implements the Detector interface from Plan 02-01.

  Language detector: Check for config files using fs.pathExists. Return LanguageInfo[] with detected languages and their config files.

  Framework detector: Read package.json (if exists), pyproject.toml (if exists). Match dependency names against a registry map of framework-name → FrameworkInfo. Return FrameworkInfo[].

  Package manager detector: Check for lockfiles. Return PackageManagerInfo or null.

  Register all three detectors in the orchestrator.
  </implementation>
</feature>
