**Registry design (from discovery):**
- Package = GitHub repo with ccxl-package.json manifest
- Manifest: { name, version, description, type, files[], keywords[], compatibility? }
- Install: fetch tarball from GitHub, extract, read manifest, copy files using merge logic
- Uninstall: read installed manifest, remove listed files
- Installed packages tracked in .claude/ccxl-packages.json
</context>

  <action>
  1. Create src/registry/install.ts:
     - async function installPackage(source: string, root: string, options: { global?: boolean }): Promise<InstalledPackage>
     - Source format: "user/repo" (GitHub shorthand) or full GitHub URL
     - Steps:
       a. Fetch tarball: GET https://api.github.com/repos/{owner}/{repo}/tarball/main
       b. Extract to temp directory
       c. Read ccxl-package.json manifest from extracted contents
       d. Validate manifest against Zod schema
       e. For each file in manifest.files: copy to target root (project .claude/ or global ~/.claude/)
       f. Use mergeWithExisting logic — skip files that already exist
       g. Record installation in .claude/ccxl-packages.json
       h. Return InstalledPackage record

  2. Create src/registry/uninstall.ts:
     - async function uninstallPackage(name: string, root: string): Promise<void>
     - Read .claude/ccxl-packages.json
     - Find package by name
     - Remove all files listed in the installed record
     - Remove package from registry
     - Save updated registry

  3. Update src/commands/install.ts:
     - Replace stub with actual implementation
     - `ccxl install user/repo` calls installPackage
     - `ccxl install user/repo --global` installs to ~/.claude/
     - `ccxl install user/repo --preview` shows what would be installed without writing
     - Print: "Installed {name}@{version} ({N} files)"

  4. Tests (using mock HTTP — create fixture tarball or mock the fetch):
     - Valid manifest installs files to .claude/
     - Install records package in ccxl-packages.json
     - --preview shows files without writing
     - Invalid manifest (missing name/version) throws error
     - Uninstall removes files and registry entry

  Note: For the GitHub fetch, use Node's built-in fetch (Node 20+). For tests, mock the fetch response with a locally-created tarball.
