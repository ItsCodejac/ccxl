<objective>
Create the skills generator that produces .claude/skills/ directory with project-appropriate skill files using proper frontmatter, triggers, and tool restrictions.

Purpose: Skills are Claude Code's extension mechanism. Project-specific skills (test runner, deployment, code review, etc.) dramatically improve the developer experience but are tedious to write by hand.
Output: Skills generator producing 5-10 project-appropriate skill files.
</objective>

**Skill frontmatter format (from research):**
```yaml
---
name: skill-name
description: What it does (max 250 chars)
user-invocable: true
allowed-tools: "Read, Grep, Glob, Bash"
model: "sonnet"
---
Body with instructions, supports $ARGUMENTS substitution
```
</context>

<task type="auto">
  <name>Task 1: Create skill template system and base skills</name>
  <files>src/generators/skills.ts, src/generators/skill-templates.ts</files>
  <action>
  1. Create src/generators/skill-templates.ts:
     Define skill templates as objects: { name, description, allowedTools?, model?, body }

     Universal skills (always generated):
     - run-tests: Runs project test suite, detects test runner from analysis
     - review-code: Code review focused on quality, security, and patterns
     - generate-tests: Generate tests for specified code
     - explain-code: Explain code architecture and patterns

     Conditional skills:
     - deploy (if Vercel/Netlify/Fly detected): Deployment workflow
     - db-migrate (if Prisma/Drizzle detected): Database migration workflow
     - lint-fix (if JS/TS detected): Run linter and fix issues
     - docker-build (if Docker detected): Build and manage Docker containers
     - ci-check (if CI/CD detected): Check CI status, debug failures

     Each template function receives ProjectAnalysis and returns the skill content string with proper frontmatter.

  2. Create src/generators/skills.ts implementing Generator interface:
     - Iterates over all skill templates
     - Evaluates conditions against ProjectAnalysis
     - Generates .claude/skills/{name}/SKILL.md for each matching skill
     - Returns GeneratedFile[] with all skill files

  Register in pipeline.
  </action>
  <verify>npx tsc --noEmit passes</verify>
  <done>Skill templates defined, generator produces conditional skill files based on analysis</done>
</task>
