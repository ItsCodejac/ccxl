  Create src/types/analysis.ts with the full analysis type system using Zod for runtime validation:

  1. LanguageInfo: { name: string, version?: string, configFiles: string[] }
  2. FrameworkInfo: { name: string, version?: string, category: 'frontend'|'backend'|'fullstack'|'build'|'test'|'orm'|'data' }
  3. PackageManagerInfo: { name: 'npm'|'yarn'|'pnpm'|'bun', lockfile: string }
  4. MonorepoInfo: { tool: string, workspaces: string[] }
  5. CIProviderInfo: { name: string, configPath: string }
  6. CloudProviderInfo: { name: string, services: string[] }
  7. DatabaseInfo: { name: string, type: 'sql'|'nosql'|'cache'|'search' }
  8. DockerInfo: { hasDockerfile: boolean, hasCompose: boolean }
  9. ExistingConfigInfo (expand from ExistingConfig): { tool, path, exists, version?: string, features: string[] }
  10. ProjectAnalysis (full): { name, root, languages, frameworks, packageManager, monorepo, ci, cloud, databases, docker, existingConfigs, analyzedAt: Date }

  Use Zod schemas to define each, then infer TypeScript types with z.infer<>. Export both schemas and types.

<task type="auto">
  <name>Task 2: Create detector interface and analyzer orchestrator</name>
  <files>src/analyzer/detector.ts, src/analyzer/analyze.ts, src/analyzer/index.ts</files>
  <action>
  1. Create src/analyzer/detector.ts:
     - Define Detector interface: { name: string, detect(root: string): Promise<DetectorResult> }
     - DetectorResult is a discriminated union per detector type (language, framework, etc.)
     - Define DetectorRegistry class that holds all detectors and runs them in parallel
     - Registry uses Promise.allSettled to run detectors — one failing detector doesn't break the whole scan

  2. Create src/analyzer/analyze.ts:
     - Export async function analyzeProject(root: string): Promise<ProjectAnalysis>
     - Accepts project root path
     - Creates DetectorRegistry, registers all detectors (empty for now — Plans 02-02 through 02-04 will add them)
     - Runs all detectors via registry
     - Assembles results into ProjectAnalysis object
     - Validates final result against Zod schema
     - Returns validated ProjectAnalysis

  3. Update src/analyzer/index.ts:
     - Export analyzeProject, Detector interface, DetectorRegistry, all types

  The detector registration pattern: each detector module exports a Detector object. The orchestrator imports and registers them. New detectors are added by importing and registering — no modification to orchestrator needed.
  </action>
  <verify>npx tsc --noEmit passes, src/analyzer/index.ts exports analyzeProject, Detector, DetectorRegistry</verify>
  <done>Detector interface defined, registry runs detectors in parallel with error isolation, orchestrator assembles and validates results</done>
</task>
