<objective>
Make ccxl non-destructive — when running on a project that already has .claude/ configs, merge new settings into existing ones instead of overwriting. Users must be able to add ccxl at any point in a project's development.

Purpose: Without merge support, running ccxl on an existing project would destroy custom settings, skills, and hooks. This is a dealbreaker for adoption.
Output: Merge-aware pipeline that respects existing configs.
</objective>

<task type="auto">
  <name>Task 1: Add merge logic to generator pipeline</name>
  <files>src/generators/merge.ts, src/generators/pipeline.ts, src/generators/types.ts</files>
  <action>
  1. Create src/generators/merge.ts:
     - async function mergeWithExisting(root: string, files: GeneratedFile[]): Promise<GeneratedFile[]>
     - For each GeneratedFile, check if file exists at root + path
     - File type strategies:
       a. JSON files (.json): deep merge — existing keys preserved, new keys added, arrays concatenated with dedup
       b. Markdown files (.md): skip if exists (don't overwrite user's CLAUDE.md)
       c. Skill files (SKILL.md): skip if skill directory already exists
       d. Shell scripts (.sh): skip if exists
       e. Agent files (.claude/agents/*.md): skip if exists
     - Update GeneratedFile.status: 'new' if doesn't exist, 'modified' if merged, 'unchanged' if skipped

  2. Special handling for settings.json merge:
     - Parse existing .claude/settings.json
     - Merge permissions: union of allow/ask/deny arrays (no duplicates)
     - Merge env: existing values take precedence
     - Merge hooks: concatenate hook arrays per event type
     - Preserve any fields ccxl didn't generate (model, effortLevel, etc.)

  3. Update pipeline.ts to call mergeWithExisting before returning files
     - Add optional `merge: boolean` parameter (default true)
     - When --force flag is used, skip merge (overwrite everything)

  4. Update GeneratedFile type to include `skipped?: boolean` for reporting
  </action>
  <verify>npx tsc --noEmit passes</verify>
  <done>Pipeline merges with existing configs, preserves user customizations, reports new/modified/unchanged</done>
</task>
