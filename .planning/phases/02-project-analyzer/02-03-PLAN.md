<feature>
  <name>Infrastructure Detection</name>
  <files>src/analyzer/detectors/ci.ts, src/analyzer/detectors/cloud.ts, src/analyzer/detectors/database.ts, src/analyzer/detectors/docker.ts, src/analyzer/detectors/monorepo.ts, src/analyzer/detectors/__tests__/infrastructure.test.ts</files>
  <behavior>
  CI/CD detector:
  - .github/workflows/ directory exists → { name: "github-actions", configPath: ".github/workflows/" }
  - .gitlab-ci.yml exists → { name: "gitlab-ci", configPath: ".gitlab-ci.yml" }
  - .circleci/config.yml → { name: "circleci", configPath: ".circleci/config.yml" }
  - Jenkinsfile → { name: "jenkins", configPath: "Jenkinsfile" }
  - .travis.yml → { name: "travis", configPath: ".travis.yml" }
  - Multiple CI configs → returns all

  Cloud provider detector (scan for SDK deps + config files):
  - aws-sdk/@aws-sdk in deps OR .aws/ dir → { name: "aws", services: [] }
  - @google-cloud in deps OR .gcloud/ dir → { name: "gcp", services: [] }
  - @azure in deps → { name: "azure", services: [] }
  - vercel.json → { name: "vercel", services: ["hosting"] }
  - netlify.toml → { name: "netlify", services: ["hosting"] }
  - fly.toml → { name: "fly", services: ["hosting"] }

  Database detector (scan deps):
  - pg/postgres in deps → { name: "postgresql", type: "sql" }
  - mysql2 in deps → { name: "mysql", type: "sql" }
  - mongodb in deps → { name: "mongodb", type: "nosql" }
  - redis/ioredis in deps → { name: "redis", type: "cache" }
  - @prisma/client → detect DB from prisma schema datasource
  - sqlite3/better-sqlite3 → { name: "sqlite", type: "sql" }

  Docker detector:
  - Dockerfile exists → hasDockerfile: true
  - docker-compose.yml or compose.yml → hasCompose: true

  Monorepo detector:
  - turbo.json → { tool: "turborepo", workspaces: [from package.json] }
  - nx.json → { tool: "nx", workspaces: [from workspace.json or package.json] }
  - pnpm-workspace.yaml → { tool: "pnpm", workspaces: [from yaml] }
  - package.json workspaces field → { tool: "npm/yarn", workspaces: [...] }
  - lerna.json → { tool: "lerna", workspaces: [from lerna.json] }
  </behavior>
  <implementation>
  Each detector implements the Detector interface. All use fs.pathExists for file checks and fs.readJson/readFile for content parsing.

  Follow the same test pattern from Plan 02-02 — mock file systems with temp directories.

  Register all 5 detectors in the orchestrator alongside the language/framework detectors from Plan 02-02.
  </implementation>
</feature>
